What you get back:

  return (

      <img src={props.image} />

Now you retry. And retry again. 3 premium requests burned — and you restart the thread each time, losing context.

Constraints: The Secret Weapon

The most underused and highest-leverage lever

Do NOT:
- Use default exports
- Use inline styles or CSS modules
- Use `any` type
- Add comments explaining basic React concepts
- Add console.log statements
- Import from barrel files

Constraints narrow the output space faster than positive instructions.

Think of it like a creative brief — telling the designer what's off-brand
is faster than describing every possible on-brand option.

  • This is the insight most people miss -- negation is more powerful than specification
  • The creative brief analogy resonates well with non-technical leaders
  • Emphasize: these constraints come from real code review feedback -- encode it once

The Engineered Prompt

You are a senior React engineer working in a TypeScript
+ Tailwind CSS codebase.

Generate a ProductCard component that:
- Accepts props: { title: string; price: number;
  imageUrl: string; isOnSale?: boolean }
- Uses Tailwind utility classes (no CSS modules)
- Includes proper TypeScript types (no `any`)
- Has hover state with scale transform
- Shows a sale badge conditionally
- Is accessible (alt text, semantic HTML, keyboard navigable)

Do NOT:
- Use inline styles
- Use default exports
- Add comments explaining basic React concepts

Return only the component file. No tests, no stories.

The APE Framework

Action → Purpose → Expectation

Vague

Fix this function

APE Prompt

Action Refactor fetchProducts to handle pagination
Purpose Users see timeouts on collections with 500+ products because we fetch everything at once
Expectation Cursor-based pagination with loading state, error recovery, using our ApiResponse<T> type

One well-structured APE prompt replaces three vague ones.

  • APE is the easiest framework to remember and apply immediately
  • Purpose is the key differentiator -- it gives the model "why" context
  • When the model knows the problem, it makes better design decisions

copilot-instructions.md

Your persistent system prompt. Write it once, the whole team benefits.

# .github/copilot-instructions.md

## Stack
- React 18 + TypeScript 5 (strict mode)
- Tailwind CSS (no CSS modules)
- Named exports only (no default exports)

## Conventions
- Arrow functions, const declarations
- Props destructured on their own line
- Zod schemas for all API boundaries

## Do NOT
- Use `any` type
- Use inline styles
- Add TODO comments
- Import from barrel index files in the same package

New developer joins → they get AI output that matches your team's standards from day one.

  • This is the scalability play -- one file, entire team benefits
  • Lives in the repo, version-controlled, reviewed like any other code
  • This is what turns individual skill into organizational capability
